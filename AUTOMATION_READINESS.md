# Automation Readiness

## Summary
- 현재 프로젝트는 정적 사이트 골격과 반응형 초기 구조는 자동화에 적합하지만, Claude Verifier의 비대화형 API 연결이 불안정해 완전 자동화는 아직 아니다.
- 가장 현실적인 운영 방식은 `AUTO`로 파일 생성/수정, `HITL`로 콘텐츠·배포 승인, `MANUAL`로 최종 배포 확인이다.

| 루프 | 자동화 수준 | Codex 역할 | Claude Verifier | 위험도 | 권한 | Rollback | 준비 상태 | 보완점 |
|---|---|---|---|---|---|---|---|---|
| 1. HTML/CSS/JS 골격 | AUTO | `index.html`, `styles.css`, `script.js` 최소 연결 | 존재/링크/viewport/메뉴 확인 | 낮음 | 로컬 파일 수정만 | 파일 단위 되돌리기 | 준비됨 | Verifier 연결 안정화 |
| 2. 반응형/테마 | AUTO | 레이아웃, 매트릭스 배경, 모바일 메뉴 조정 | 375/768/1440px 확인 | 중간 | 로컬 파일 수정만 | CSS 단위 되돌리기 | 부분 준비 | 화면별 스냅샷 검증 추가 |
| 3. 게임 기능 | HITL | 지렁이 게임 입력/충돌/상태 수정 | 키보드·터치·게임 루프 확인 | 높음 | 구현 변경 승인 필요 | 게임 파일 단위 되돌리기 | 미준비 | 입력 시나리오 정의 필요 |
| 4. 콘텐츠/CV | MANUAL | CV 반영, `[사람 확인 필요]` 정리 | 콘텐츠 정확성 확인 | 높음 | 사람 승인 필요 | 콘텐츠 덮어쓰기 금지 | 미준비 | CV 원문 확보 필요 |
| 5. 배포 | MANUAL | GitHub Pages 설정 점검 | 배포 후 공개 URL 확인 | 높음 | 배포 승인/토큰 사용 | 배포 전 태그/커밋 기준 복구 | 미준비 | 배포 전 체크리스트 필요 |

## 지금 자동화하기 가장 좋은 루프 1개
- 루프 1: HTML/CSS/JS 골격
- 이유: 파일 존재, 링크, viewport, 메뉴 같은 정적 검증은 자동화 친화적이고 실패 원인이 명확하다.

## 사람이 반드시 승인해야 하는 지점
- CV 원문이 없을 때 프로필 문구를 채우는 순간
- 지렁이 게임의 규칙 해석이 바뀌는 순간
- GitHub Pages 배포와 토큰 사용이 필요한 순간
- 기존 콘텐츠를 삭제하거나 큰 구조를 바꾸는 순간

## Claude Verifier가 확인해야 할 항목
- 파일 존재와 상대 경로
- HTML 구조와 내부 링크
- CSS 반응형: `375px`, `768px`, `1440px`
- JavaScript 오류와 메뉴 토글
- 지렁이 게임 입력과 루프
- 로컬 HTTP 응답
- GitHub Pages 호환성

## Claude 사용 불가 시 Fallback 방식
- `CODEX_FALLBACK`으로 전환한다.
- Codex가 수정과 로컬 확인을 모두 수행한다.
- 실패 기록은 `AORR_LOG.md`에 남긴다.
- `Claude API connection refused` 또는 timeout이 반복되면 HITL로 올린다.

## 운영 전 필요한 테스트·권한·모니터링
- 테스트: 파일 존재, 링크, viewport, 반응형, 입력, 로컬 실행
- 권한: 로컬 파일 쓰기, Git 상태 확인, 배포 시 GitHub Pages 권한
- 모니터링: Claude 연결 상태, 실패 fingerprint, 변경 파일 범위, 배포 URL

## 현업 적용 다음 액션 3개
1. Claude Verifier를 새 세션에서 `--print` 없이 안정적으로 붙는지 재확인한다.
2. 게임 루프 전 최소 단위로 지렁이 게임 입력 뼈대를 만든다.
3. CV 원문을 확보해 ` [사람 확인 필요] ` 항목을 실제 내용으로 교체한다.
