# Step 1 개발 업무 분석

## 전제 확인
- 현재 작업공간에서 실제 CV 파일은 확인되지 않았습니다.
- `강의교안.pdf`만 보이며, 이것이 CV인지 여부는 확인되지 않았습니다.
- 따라서 경력/프로필 기반 내용은 모두 ` [사람 확인 필요] `로 분리합니다.
- 토큰 값은 읽거나 저장하지 않고, 파일명만 기록합니다.

## 프로젝트 반영 범위
- 정적 `HTML`, `CSS`, `JavaScript`만 사용합니다.
- 반응형 개인 프로페셔널 웹사이트를 만듭니다.
- 상단 `Games` 메뉴를 추가합니다.
- 키보드와 모바일 터치로 조작하는 지렁이 게임을 넣습니다.
- 메트릭스 느낌의 움직이는 배경을 적용합니다.
- 게임 확장 기능으로 랜덤 이동 적 5개, 5초마다 폭발, 폭발 후 랜덤 재생성, 무한 반복을 반영합니다.

## CV/프로필 반영 원칙
- 이름, 경력, 학력, 스킬, 프로젝트, 연락처 등은 CV 확인 전까지 ` [사람 확인 필요] `로 둡니다.
- 자기소개 문구와 하이라이트는 CV가 확인되면 그때 교체합니다.
- 현재는 레이아웃과 데이터 바인딩 자리만 설계합니다.

## 가장 안전한 첫 루프 추천
- 추천 루프: `1`
- 이유: 아직 CV가 확인되지 않았고, 게임 규칙도 추가 요구가 많아서 먼저 정보 구조와 미확정 항목 경계를 잡는 것이 가장 안전합니다.
- 이 루프에서는 실제 디자인이나 구현보다, 어떤 데이터를 어디에 넣을지와 무엇을 ` [사람 확인 필요] `로 둘지 먼저 정합니다.

## 1차 실행 계획
1. 사이트 섹션을 정의합니다.
2. CV 미확인 항목을 전부 ` [사람 확인 필요] `로 표기합니다.
3. `Games` 메뉴와 지렁이 게임 진입 위치를 구조에 포함합니다.
4. 매트릭스 배경과 적 5개 순환 로직은 다음 루프에서 구현 대상으로 분리합니다.

## 위험 및 확인 필요
- CV 내용이 없어서 개인 프로필 문구는 아직 확정할 수 없습니다.
- 로컬에 보이는 토큰 파일명은 `github_token.txt.txt`이지만, 요청된 기록명은 `github_token.txt`입니다. 실제 사용 전 파일명 일치 여부는 ` [사람 확인 필요] `입니다.
- 코드 수정, 테스트, push, 배포는 이번 단계에서 수행하지 않습니다.

## 확인 필요 목록
- 실제 CV 파일 여부
- CV에 들어갈 이름
- CV에 들어갈 경력
- CV에 들어갈 학력
- CV에 들어갈 스킬
- CV에 들어갈 프로젝트
- CV에 들어갈 연락처
- `github_token.txt`와 실제 로컬 파일명 `github_token.txt.txt`의 일치 여부
